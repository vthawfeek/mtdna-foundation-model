Generate a daily HTML dashboard for the current project and open it in the browser.

## Context

Working directory: current project directory.
Output: `dashboard.html` in the project root.
Day content (ELI5, Math, Details) lives in `reports/day-N-data.js` files, generated by
`/day N`. This skill generates only the scaffold — no per-day content is written here.

## Steps

### 1. Extract project data

Read `CLAUDE.md` and find all lines in the "Current status" section. Parse them to find:
- `LAST_DAY`: the highest day number that has `COMPLETE` in its line
- `LAST_DAY_TOPIC`: the topic name from that line
- `LAST_DAY_COMMIT`: the commit hash from that line (e.g. `commit 12d614c`)
- If no days are complete, set `LAST_DAY = 0`

Set `TODAY_DAY = LAST_DAY + 1`.

Read `PLAN.md` and locate `### Day TODAY_DAY:` and `### Day (TODAY_DAY+1):`. Extract:
- `TODAY_TOPIC`: the heading text after "Day N:"
- `TODAY_TASKS`: all bullet points under that heading (stop at next `###` or `---`)
- `NEXT_TOPIC`: heading text of the following day
- `NEXT_TASKS`: first 3 bullet points of the following day

Read `reports/day-LAST_DAY-*.md` (glob — pick the matching file):
- Extract everything under `## What was learned` up to the next `##` section
- Call this `YESTERDAY_LEARNINGS` (list of bullet points, max 6)

Check if `reports/day-TODAY_DAY-*.md` exists:
- If yes: extract `## What was learned` bullets → `TODAY_LEARNINGS`
- If no: `TODAY_LEARNINGS = []`, set `TODAY_NOT_STARTED = true`

Check for pending social media by scanning `reports/` for files matching `day-*-blog.md`:
- For each, check if it contains a line starting with `<!-- published:` → if not, it is pending
- Build a list: `[{day: N, command: "/blog-send N"}, ...]`

Calculate `PROGRESS_PCT = round(LAST_DAY / 28 * 100)`.

Determine the week label for TODAY_DAY:
- Days 1-7: "Week 1: Infrastructure and Data"
- Days 8-14: "Week 2: Model Architecture &amp; Pre-training"
- Days 15-21: "Week 3: Fine-tuning and Evaluation"
- Days 22-28: "Week 4: Production Polish"

### 2. Find available day data files

List all files matching `reports/day-*-data.js` in the project directory, sorted
numerically by day number. Extract the day numbers. Call this `DATA_DAYS`.

For each day 1–28: set `status` to:
- `"comp"` if day ≤ LAST_DAY
- `"tod"` if day == TODAY_DAY
- `""` (future/locked) otherwise

### 3. Compute recommended posting dates

From today's date:
- `X_POST_DATE`: next weekday (Mon-Fri), formatted as "Mon 25 May"
- `LI_POST_DATE`: next Tuesday, Wednesday, or Thursday — whichever comes first

### 4. Write dashboard.html

Write a complete, self-contained HTML file. The `<script src>` tags for day data files
are injected in the `<head>` — one per file in DATA_DAYS, sorted numerically. The JS
then reads from `window.__dayData` (populated by those files) when a day is clicked.

Use this exact structure (substitute all `{PLACEHOLDER}` values with real data):

// dayData is populated by the per-day <script src="reports/day-N-data.js"> tags in <head>
const dayData = window.__dayData || {};

// Days that have a data file available (populated from script tags above)
const dataDays = new Set(Object.keys(dayData).map(Number));

const LAST_DAY = {LAST_DAY};
const TODAY_DAY = {TODAY_DAY};

let activeDay = null;

function renderNavButtons() {
  const container = document.getElementById('nav-btns');
  for (let d = 1; d <= 28; d++) {
    const btn = document.createElement('button');
    btn.className = 'db';
    btn.textContent = 'D' + d;
    btn.dataset.day = d;
    if (d <= LAST_DAY) {
      btn.classList.add('comp');
      btn.onclick = () => showDay(d);
    } else if (d === TODAY_DAY) {
      btn.classList.add('tod');
      if (dataDays.has(d)) btn.onclick = () => showDay(d);
    }
    container.appendChild(btn);
  }
}

function showTab(tabId, btn) {
  document.querySelectorAll('#day-panel .tab-pane').forEach(p => p.classList.remove('active'));
  document.querySelectorAll('#day-panel .pt-btn').forEach(b => b.classList.remove('active'));
  document.getElementById(tabId).classList.add('active');
  btn.classList.add('active');
}

  document.querySelectorAll('.db').forEach(b => b.classList.remove('active'));

  if (btn) btn.classList.add('active');

- `{W1_CUR}` through `{W4_CUR}`: set to ` cur` (with leading space) for the active week, empty string otherwise
- `{FOR EACH}` / `{IF}` blocks: render as actual HTML with real values, not template tags
- `<script src="reports/day-{day_num:02d}-data.js">`: zero-pad day number to 2 digits (01, 02, …, 15)
- `const LAST_DAY = {LAST_DAY};` and `const TODAY_DAY = {TODAY_DAY};` in the JS block: substitute the actual numbers
